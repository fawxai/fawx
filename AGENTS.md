# AGENTS.md

This repository's authoritative project guidance lives in these files:

- `ENGINEERING.md`
- `DOCTRINE.md`
- `TASTE.md`

Required behavior:

- Read and follow these documents before making substantial changes.
- Treat `ENGINEERING.md` as immutable engineering doctrine.
- Treat `DOCTRINE.md` as immutable runtime/kernel doctrine.
- Treat `TASTE.md` as evolving judgment and design guidance for cases where doctrine does not fully prescribe the answer.
- Do not weaken, bypass, or silently reinterpret `ENGINEERING.md` or `DOCTRINE.md`.
- If there is tension between these files, precedence is:
  1. `ENGINEERING.md`
  2. `DOCTRINE.md`
  3. `TASTE.md`

In short: doctrine is binding, taste guides judgment, and all three files are part of the default instruction surface for work in this repo.
