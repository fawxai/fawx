you › What's the difference between a mutex and a semaphore?

assistant › A mutex (mutual exclusion) and a semaphore are both synchronization primitives used in 
concurrent programming, but they have some key differences:

1. Ownership:
- Mutex: Has strict ownership - only the thread that locks a mutex can unlock it
- Semaphore: Has no concept of ownership - any thread can increment or decrement it

2. Counting:
- Mutex: Is binary (locked/unlocked) - can only be locked once
- Semaphore: Can have multiple values and can be incremented/decremented by different 
threads

3. Purpose:
- Mutex: Used primarily for protecting shared resources (mutual exclusion)
- Semaphore: Used for signaling between threads and managing resource pools

4. Usage:
- Mutex: Best for exclusive access scenarios (like protecting shared data)
- Semaphore: Better for producer-consumer scenarios or limiting resource usage

5. Operations:
- Mutex: lock() and unlock()
- Semaphore: wait() (or P()) and signal() (or V())

Think of a mutex like a single key to a room - only one thread can have the key and enter
at a time. A semaphore is more like a parking lot with multiple spaces - it can handle 
multiple threads up to a set limit.
