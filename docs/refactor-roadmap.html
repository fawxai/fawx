Kernel contract stabilization first. One PR at a time on dev. New substrate and authority work resume only after the loop, transcript, and persistence contracts are trustworthy again.

Checkpoint (2026-03-29): The issue set since #1637 shows the real shape of the work. We are not in feature expansion mode. We are in kernel stabilization mode. #1639 landed and gives us a better tool spine, but commit tracing split current instability into three separate runtime problems: #1641 direct-inspection fake-permission failure, #1658 runaway continuation reintroduced by 13376838, and #1656 post-turn unknown-tool / ordering corruption. The plan below treats those as first-class gates.

Linear plan we can begin now

Step 1. #1641 Phase 1: direct-inspection detection
Route deterministic local inspection requests into typed profile state instead of Standard.
Step 2. #1641 Phase 2: observation-only tool surface
Own the allowed tool set inside the profile. No inheritance from the broad standard continuation contract.
Step 3. #1641 Phase 3: terminal completion from evidence
End the turn on the first usable post-tool answer. This is the key gate against both fake permission denial and direct-inspection flailing.
Step 4. #1641 Phase 4: standard/profile boundary cleanup
Remove remaining leakage between Standard and direct inspection. Lock the behavior in with regression tests.
Step 5. #1658: mixed-text runaway follow-up
Immediately retest after Step 4. If #1641 fully subsumes the direct-inspection runaway, narrow this issue to remaining mixed-text cases. If not, fix the reintroduced continuation loop directly.
Step 6. #1656: enforce tool ordering and reject poisoned history
Fix the session write boundary before doing any history UX cleanup.
Step 7. #1655: persist one grouped tool-history record per turn
Turn-scoped transcript contract. This comes after #1656, not before.
Step 8. #1646: direct utility routing and visible tool contracts
Resume substrate work once the runtime can finish and persist deterministic turns cleanly.
Step 9. #1647: revisioned skill activation lifecycle
Build on #1646 and the landed tool registry.
Step 10. Unified authority model implementation
Spec is already written. Implementation begins after runtime and substrate contracts are stable.

Execution gates for the stabilization tranche

  • Gate A: Read the file at ~/.zshrc and explain what each line does must not end in an outside-working-directory refusal once the file was successfully read.
  • Gate B: that prompt must terminate without runaway continuation on the current direct-inspection path.
  • Gate C: no post-turn Unknown tool call row for the traced prompt after completion.
  • Gate D: persisted history must never store tool_result before matching tool_use.
  • Gate E: one completed turn must replay as one grouped historical tool-activity unit.
