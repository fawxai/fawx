Core Engine

Single Rust binary. Single-pass agentic loop with tool calling, memory, and multi-provider support. Hard boundaries via TerminationConfig and ToolRetryTracker.

  • ✓ Single-pass agentic loop with multi-tool calls (verify/learn/should_continue removed)
  • ✓ TerminationConfig: nudge at 6 tool-only turns, strip tools at 9
  • ✓ ToolRetryTracker with no-progress detection and per-signature failure tracking
  • ✓ 13+ built-in tools (files, web, browser, code execution)
  • ✓ Persistent memory with semantic search
  • ✓ Multi-provider (Claude, GPT, local models)
  • ✓ Encrypted credential store (AES-256-GCM)
  • ✓ Config management with hot-reload (SIGHUP restart)
  • ✓ Token-by-token streaming over SSE
  • ✓ Reflective memory (fx-journal crate)
  • ✓ Conversation compaction with tool pair integrity preservation
  • ✓ Composable system prompt architecture
  • ✓ Graceful loop termination with forced synthesis
  • ✓ Chat attachments: images, text files, PDFs (provider-native rendering)
  • ✓ 1,800+ tests, clippy clean

AX-First Kernel Safety

Agent Experience first. Capability space up front, tripwire/ripcord inside the boundary, OS-level enforcement outside it. The kernel is an optimization layer, not a checkpoint system.

  • ✓ CapabilityMode (default): immediate structured denial for restricted actions
  • ✓ Prompt mode preserved as opt-in
  • ✓ Presets: Open / Standard / Restricted
  • ✓ Tripwire: silent boundary monitoring within capability space
  • ✓ Ripcord: atomic rollback of file/git operations from tripwire point
  • ✓ request_capability tool for agent-initiated escalation
  • ✓ OS-level sandbox (fx-sandbox: Landlock filesystem, seccomp-bpf syscalls, network policy)
  • ✓ Tier-3 immutable paths (compiled into binary)
  • ✓ Kernel manifest for structured self-introspection
