# Fawx Native App — Cowork Mockup Prompt

**Purpose:** Generate pixel-accurate HTML mockups of every screen in the Fawx native app. These will be screenshotted and provided as visual reference images to Codex, which will build the actual SwiftUI implementation. Accuracy matters — every color, spacing value, and layout decision flows directly into code.

**Reference documents (attached or pasted alongside this prompt):**
1. `docs/specs/swift-app-spec.md` — the APPROVED implementation spec (v5, R7 APPROVE). This is the source of truth for all behavior, layout, and API contracts.
2. `docs/design/design-system.html` — interactive design system with all components rendered (dark/light toggle).

**Output:** A single HTML file with tabbed navigation across all screens. Each screen should be screenshot-ready at native resolution. Include a dark/light mode toggle in the top corner — every screen needs both modes captured.

## Rendering Rules

1. **Dark mode is the default view.** Light mode via toggle in top corner.
2. **macOS screens** render at ~1200×800 viewport. Simulate macOS window chrome (traffic light buttons ● ● ●, title bar).
3. **iOS screens** render in an iPhone-shaped frame (390×844, rounded corners, notch/dynamic island at top, home indicator at bottom).
4. **Code blocks are ALWAYS monochrome** — single text color on code background. No syntax highlighting. This is intentional and matches the V1 spec.
5. **Status bar model text is NOT interactive** — render as plain text, no underline, no pointer cursor.
6. **Hover states** should work on macOS screens (sidebar items, buttons).
7. **Tabs at top of the HTML page** to navigate between all 9 screens.
8. **Each screen must be independently screenshot-able** — no overlapping, no scrolling needed for a single screen. If content is tall, show a realistic viewport with scroll indication.
9. Use CSS variables for all colors/spacing so the dark/light toggle actually works everywhere.
10. **Fawx accent orange (#E8711A dark / #D45E14 light) should be prominent** — buttons, selected states, accents. This is the brand color.

## What These Mockups Will Be Used For

These HTML screens will be screenshotted (dark + light for each = 18 images) and provided to Codex as visual reference for building the SwiftUI app. Codex will have:
1. These screenshot images (for pixel reference)
2. The approved `swift-app-spec.md` (for behavior/architecture/API contracts)

The mockups don't need to be functional (no real API calls). They need to be **visually accurate** — correct colors, correct spacing, correct typography, correct layout proportions. Codex will translate pixels into SwiftUI code.
