Strategic Vision

Every existing agentic harness (Codex CLI, Claude Code, OpenClaw, Hermes Agent) has the same fundamental limitation: they're terminal-first tools bolted onto chat interfaces. None of them have a native GUI, a real security kernel, or a self-improvement loop.

Fawx wins by being the first harness that is simultaneously:

🖥 Native GUI

macOS + iOS Swift app. No terminal required. Session management, streaming visualization, tool approval, memory browsing. Zero competitors have this.

🔒 AX-First Security

Capability gates (not consent prompts), tripwire/ripcord invisible enforcement, OS-level sandboxing. The agent runs free inside boundaries; the human sets policy, not individual approvals.

🧠 Self-Improving

fx-forge: experiment → evaluate → fine-tune loop. Signal flywheel from tripwire data. The agent gets better from its own mistakes, automatically.

⚡ Fleet Architecture

Multi-node orchestration over Tailscale. Subagent spawning, task routing, distributed tool execution. One brain, many hands.

5 Critical Gaps to Close

1. Structured Session Storage

Tool calls are flattened to text. The model can't see what tools it ran or what they returned. Context compaction loses critical information. This is why long sessions degrade.

Fix: PR C — structured message types in redb, tool call/result pairs preserved through compaction.

engine

2. Response Quality Gate

Tool errors surface as user-facing text. Model parrots error messages instead of synthesizing a real response. Users see raw stack traces and git errors.

Fix: PR B — extract_response_text classifies outputs, routes errors back through synthesis.

engine

3. Memory Continuity

fx-journal exists but isn't wired like OpenClaw's MEMORY.md system. No soul import path. No cross-session memory search. No memory decay or curation.

Fix: Memory format compatible with import, semantic search over journal entries, auto-curation.

engine ax

4. Multi-Turn Autonomy

Agent should run 50+ tool calls without user intervention. Currently limited by timeout, context window, and lack of checkpoint/resume.

Fix: Longer timeouts (done: 30 min), checkpoint-based context management, graceful degradation on compaction.

engine ax

5. Self-Improvement Loop

fx-forge experiment pipeline works but isn't connected to the signal flywheel. No automatic skill refinement from tripwire data. No earned autonomy.

Fix: Wire tripwire signals → experiment generation → evaluation → skill update. Forge Phase 2.

ax
