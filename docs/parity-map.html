Recommended Skill Build Order

1
edit tool — surgical file editing Engine
Replace exact text in files without rewriting the whole file. Core ergonomic need — every coding agent has this. Add offset/limit to read_file for large file support. Engine-level, not a WASM skill.
2
Background process management Engine
Extend run_command with background execution, session IDs, poll/log/kill. Needed for long-running builds, dev servers, test suites. Engine-level — WASM can't manage host processes.
3
Vision skill — image analysis WASM Skill
Send images to a VLM (GPT-4o, Claude, Gemini). Accepts file path or URL, returns analysis. Spec exists (#1016). WASM skill using Network capability + provider API.
4
TTS skill — text-to-speech WASM Skill
Convert text to audio. Edge TTS (free), ElevenLabs, OpenAI providers. Spec exists (#1017). WASM skill using Network + FileSystem capabilities.
5
Browser skill — web automation WASM Skill
Headless browser control. Could delegate to a Playwright process or use a simpler HTTP-based approach. Heaviest to implement — may be WASM + subprocess hybrid.
6
Memory skill — semantic search WASM Skill
Add embedding-based semantic search to complement Fawx's existing CRUD memory. Could use local embeddings (nomic) or API-based. Improves recall quality over keyword matching.
7
Canvas skill — UI rendering WASM Skill
Render HTML/data visualizations for the user. Lower priority — Fawx is CLI-first. Useful for reports, dashboards, data exploration.
