### Thinking Registry — Ship Blocker Analysis (16:48 UTC)
Joe confirmed thinking is broken, can't use xhigh on OpenAI. Root cause analysis:

**What's wrong:**
1. **OpenAI**: `reasoning_effort` completely missing from `ResponsesRequestBody`. The WebSocket Responses API needs `reasoning: { effort: "xhigh" }` but Fawx sends nothing. All OpenAI models run at provider default.
2. **Anthropic 4.6**: `ThinkingConfig::Enabled { budget_tokens }` is **deprecated**. Should use `thinking: { type: "adaptive" }` + `effort` parameter. No effort field exists in the code at all.
3. **Missing GPT-5.4**: only `gpt-5` prefix match exists, no specific GPT-5.4 entry.

**Correct API payloads per provider (from docs):**

Anthropic 4.6 (Opus/Sonnet):
```json
{ "thinking": { "type": "adaptive" }, "effort": "high", "max_tokens": 16000 }
```
Effort levels: low, medium, high (default), max (Opus 4.6 only)

Anthropic 4.5 and older (legacy):
```json
{ "thinking": { "type": "enabled", "budget_tokens": 10000 }, "max_tokens": 16000 }
```

OpenAI Responses API (WebSocket):
```json
{ "reasoning": { "effort": "xhigh" } }
```
GPT-5: minimal, low, medium (default), high
GPT-5.2: none (default), low, medium, high
GPT-5.4: none (default), low, medium, high, xhigh

- Anthropic subscription setup-token auth temporarily disabled (was broken yesterday)
- API key auth works for both providers
